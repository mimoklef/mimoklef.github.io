Moocs

I made this project for my first intership in summer 2018.

It was my first real web application and it took me a little bit less than one month to learn PHP for the first time and to create the app.

The objective of this application is to give teachers at the university the ability to book a recording room in which they record their lessons in the form of Moocs.

PS : Don't pay attention to the design it has been changed for the final version of the application (deployment).

Used technologies are PHP, MySQL and web technologies as HTML, CSS, and JavaScript/JQuery.

A few features from this app


User previlieges

It has two account types, admins and members.
Admins can access some pages that members can't reach.
As you can see in the pictures here, the nav bar is different depending on the account type.

And of course in those pages the first PHP statement is a condition that verifies user previlieges and displays a message if the user isn't a admin.

Profiles

- All users have a profile page that everyone can visit. This page contains a search form to search for users by their usernames.
- Users can also upload a picture to use as profile picture.

Mailbox


Users can send mails to each others and access their mailboxes.

Member counter

In the footer. Yeah like in the old forums.


Reservations

This one is the most important feature of this project.

Users can create reservation requests that admins can validate or not.

PS : Yeah I know the design is too simple, it was my first web app ever so ¯\_(ツ)_/¯


Here is the reservation requests page that only admins can access.

Time slot availability check

This one is the second most important feature.

Users can check if a time slot is available or or not.

This is also automatically done when they make a new reservation request so they can't create it if it overlaps another existing one.


And if there is a problem they can contact an admin by pressing the button in the bottom.

Moocs

J'ai réalisé ce projet pour mon premier stage en été 2018.

C'était ma première véritable application web et il m'a fallu un peu moins d'un mois pour apprendre le PHP pour la première fois et pour créer l'application.
L'objectif de cette application est de donner aux enseignants de l'université la possibilité de réserver une salle d'enregistrement dans laquelle ils enregistrent leurs cours sous forme de Moocs.

PS : Ne faites pas attention au design, il a été modifié pour la version finale de l'application (déploiement).

Les technologies utilisées sont PHP, MySQL et les technologies web basiques comme HTML, CSS, et JavaScript/JQuery.

Quelques fonctionnalités de l'application


Droits

Il existe deux types de comptes, les admins et les membres.
Les administrateurs peuvent accéder à certaines pages inaccessibles aux membres .
Comme vous pouvez le voir sur les photos ici, la barre de navigation est différente selon le type de compte.

Et bien sûr, dans ces pages, le premier bloc PHP est une condition qui permet de vérifier les droits de l'utilisateur et affiche un message si l'utilisateur n'est pas un admin.

Profils

- Tous les utilisateurs ont un profil que tout le monde peut visiter. Cette page contient un formulaire de recherche pour rechercher les utilisateurs par leur nom.
- Les utilisateurs peuvent également télécharger une photo à utiliser comme photo de profil.

Messagerie


Les utilisateurs peuvent s'envoyer des messages sous forme de petits mails.

Compteur de membres

Dans les footers. Ouais comme dans les vieux forums.


Réservations

C'est la fonctionnalité la plus importante du projet.

Les utilisateurs peuvent créer des demandes de réservation que les admins peuvent valider ou pas.

PS : Oui le design est très simple, mais c'était ma toute première application web ¯\_(ツ)_/¯


Voici la page de requêtes de réservations que seuls les admins peuvent voir.

Contrôle de la disponibilité des créneaux horaires

C'est le deuxième fonctionnalité la plus importante du projet.

Les utilisateurs peuvent vérifier si un créneau horaire est disponible ou pas.

Cela se fait aussi automatiquement lorsqu'ils font une nouvelle demande de réservation, de sorte qu'ils ne puissent pas la créer si elle empiète sur une autre existante.


Et s'il y'a le moindre problème ils peuvent contacter les admins en cliquant sur le bouton en bas.
