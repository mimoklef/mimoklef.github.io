Weather App

I made this project for my first Node.js class.

It took me a little bit less than two weeks (while having other courses) to create the app.

This is a simple weather web application. Users can register to save their cities, they can access realtime weather and forecast.

Main used technologies are Node.js, Express.js, simple HTML and CSS and JQuery.

A brief presentation


Application architecture

-Only authentified users can access the application.
-Users can eiter login on create a new account, they can store their favorite cities to have access to their current weather in Home page and forecast in detail page.
-Weather is gotten from OpenWeather API.
-Data is stored in a MongoDB database.
-The app uses many security protections

User's cities

Users can add, modify and delete cities that they want to see in their home menu.

To add a city, they need to open a form that looks like this :

This form has an auto completion feature to make sure users can't add fictive cities, or simply cities that aren't managed by the API.

First of all, the user needs to select a country.

Then according to the country he choosed, he can chose a city.

The way it works is pretty simple :

Countries list is directly loaded after the page finishes loading, it's a simple static JSON file than we load in client-side using JavaScript and AJAX.
When the user selects a city, the value is directly used as a parameter to load cities list from the API using AJAX.
The API has a route that returns a list of all supported cities of a country in JSON format.

Then the list is saved in client-side to be used in the auto completion.

Security

To make this app as secure as it can be, I used many protections :
-HTTPS
-JWT
-CSRF Tokens
-Rate limit (to prevent DDOS)
-XSS filter
-Password hash (Bcrypt)
-Helmet js (HTTP headers)
